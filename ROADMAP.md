# Fact Card System — Roadmap

## Context

Проект — side-project с потенциалом. MVP рабочий: WebSocket, Auth0, AI-guided flow, multi-session, экспорт в MD/PNG. Цель roadmap — систематизировать задачи для развития продукта от "работает для себя" до "можно показать людям и проверить спрос".

---

## 1. Переделать лендинг

**Проблема:** Текущий лендинг выглядит аккуратно, но плохо доносит ценность. Позиционирует продукт как пассивный визуализатор ("решение проявится само"), хотя на деле это AI co-reasoning система с методологией. Нет упоминания AI, нет конкретного use case, нет демонстрации процесса. Человек не понимает, зачем ему это.

**Решение:** Новый лендинг с чётким value proposition: конкретная боль → как работает → что получишь. Показать процесс (before/after или анимация). Один use case, один CTA. Использовать Gemini для копирайта, Tailwind v4 + компонентный подход.

**Задачи:**
- [x] Написать новый копирайт через /gemini-copywriter (RU + EN)
- [x] Спроектировать структуру лендинга (hero + как работает + результат + CTA)
- [x] Сверстать через /frontend-design с Tailwind v4
- [ ] Добавить визуальную демонстрацию процесса (скриншоты или анимация)
- [ ] A/B: подготовить 2 варианта headline для теста

## 2. Онбординг первого пользователя

**Проблема:** Новый пользователь попадает на пустой канвас и не понимает, что делать. Нет обучения, нет примера. OnboardingOverlay существует, но минимальный.

**Решение:** Guided first session — либо интерактивный тур, либо пример-шаблон, который можно пройти за 5 минут и понять механику.

**Задачи:**
- [ ] Создать демо-сессию (пример проблемы с 5-7 карточками)
- [ ] Добавить пошаговый тур: "напиши проблему → AI создаст карточки → перетащи → увидь паттерн"
- [ ] Добавить подсказки-тултипы при первом использовании

---

## 3. Экспорт артефакта (decision memo)

**Проблема:** Результат сессии живёт только на канвасе. Пользователь не получает ощутимый output, который можно переслать, сохранить, вернуться к нему. Текущий MD-экспорт — плоский список карточек без структуры.

**Решение:** Генерировать структурированный документ после сессии: проблема → ключевые факты (по кластерам) → боли → ресурсы → выявленные пробелы → предложенные действия.

**Задачи:**
- [ ] Спроектировать формат decision memo (MD)
- [ ] Добавить AI-генерацию summary по завершении сессии
- [ ] Реализовать экспорт memo (скачивание + копирование)

---

## 4. Обнаружение "пустых зон" (bliоnd spots)

**Проблема:** Ключевая фишка методологии Курпатова — найти то, чего НЕТ на карте. Сейчас AI задаёт уточняющие вопросы, но не анализирует пробелы системно. Нет визуальной индикации "пустых зон".

**Решение:** После накопления 10+ карточек AI анализирует, каких аспектов не хватает, и подсвечивает это пользователю (вопросы + визуальные маркеры на канвасе).

**Задачи:**
- [ ] Добавить gap analysis prompt в AI service (после фазы FACTS)
- [ ] Реализовать UI для "пустых зон" (placeholder-карточки или подсветка зон)
- [ ] Добавить специальные вопросы для пробелов: "А где здесь вы?", "Что бы сказал X?"

---

## 5. Мобильная версия

**Проблема:** Базовая адаптивность есть (breakpoint 768px, drawer), но drag-and-drop, создание карточек double-click, и zoom не адаптированы для touch. Канвас неудобен на телефоне.

**Решение:** Для мобильной версии упростить интерфейс: фокус на чат + список карточек, а не полноценный канвас. Канвас — для десктопа.

**Задачи:**
- [ ] Определить mobile UX: чат-first или упрощённый канвас
- [ ] Реализовать pinch-to-zoom
- [ ] Адаптировать создание карточек для touch (long press вместо double-click)
- [ ] Тестировать на реальных устройствах

---

## 6. Улучшить AI quality

**Проблема:** AI prompt использует хардкод координат (px), нет валидации AI output, температура не тюнилась, merge-логика неточная. На мобильных устройствах карточки могут размещаться за экраном. AI слишком часто перезаписывает входную (root) карточку пользователя — меняет формулировку проблемы, которую человек сам задал.

**Решение:** Нормализовать координаты (0-1), добавить валидацию AI-ответов, тюнить промпт на реальных сессиях. Защитить входную карточку от частых перезаписей — AI должен уточнять, а не переформулировать за пользователя.

**Задачи:**
- [ ] Перейти на нормализованные координаты (0-1) вместо пикселей в AI prompt
- [ ] Добавить валидацию AI output (bounds check, duplicate check)
- [ ] Защитить root-карточку: AI не должен менять текст входной карточки без явного запроса
- [ ] Собрать 10+ реальных сессий и оценить качество AI-ответов
- [ ] Настроить temperature на основе данных

## 7. TODO карточки как action plan

**Проблема:** AI генерирует TODO-карточки, тип существует в модели, но в UI они никак не выделены. Нет фильтрации, нет отдельного экспорта (i18n ключи есть, но фича не реализована).

**Решение:** Вывести TODO как отдельный слой — чеклист действий, который появляется после сессии.

**Задачи:**
- [ ] Добавить фильтр/панель TODO-карточек
- [ ] Реализовать toggle done/undone для TODO
- [ ] Отдельный экспорт TODO-листа (уже есть i18n ключи)

## 8. Валидация спроса

**Проблема:** Нет данных о том, нужен ли продукт кому-то кроме автора. Отчёт IdeaKiller основан на placeholder-цифрах.

**Решение:** Минимальная проверка: дать 10 людям попробовать, собрать обратную связь.

**Задачи:**
- [ ] Провести 5-10 сессий самому (разные проблемы)
- [ ] Дать попробовать 5-10 знакомым (наблюдать, где застревают)
- [ ] Добавить простую аналитику: сколько карточек за сессию, длительность, возврат
- [ ] Собрать feedback-форму (Google Form или typeform)

## 9. Надёжность авторизации Google

**Проблема:** Авторизация через Auth0/Google может быть нестабильной — токены истекают, refresh token может не обновляться, пользователю приходится перелогиниваться. Недавний фикс (78fd0b4) уже чинил expired refresh token, но нужно убедиться, что flow полностью надёжен и не требует пересоздания сессии.

**Решение:** Проверить весь auth flow end-to-end: логин → работа → закрытие вкладки → возврат через день → продолжение работы без повторного логина. Исправить проблемы если есть.

**Задачи:**
- [ ] Проверить auth flow: логин → закрыть → открыть через 24ч → сессия жива
- [ ] Убедиться что refresh token работает корректно (не требует re-login)
- [ ] Проверить edge cases: истёкший token, отозванный доступ, смена пароля Google
- [ ] Добавить graceful fallback: если auth сломался → понятное сообщение + кнопка re-login

## 10. Промо-статья (Reddit / блог / LinkedIn)

**Проблема:** Продукт никто не знает. Даже если лендинг будет отличный — без трафика это бесполезно. Нужен контент, который привлечёт первых пользователей и даст обратную связь.

**Решение:** Написать промо-статью с демонстрацией метода на реальном примере. Опубликовать на Reddit (r/productivity, r/selfimprovement, r/DecisionMaking), LinkedIn, и/или личном блоге. Не продавать — показывать метод и давать попробовать.

**Задачи:**
- [ ] Провести показательную сессию и задокументировать процесс (скриншоты, before/after)
- [ ] Написать статью через /gemini-copywriter: "Как я разложил сложное решение на факты за 30 минут"
- [ ] Адаптировать под формат Reddit (короче, неформальнее) и LinkedIn (профессиональнее)
- [ ] Опубликовать, собрать feedback и трафик
